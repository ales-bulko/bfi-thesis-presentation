Katedra počítačov a informatiky

Problem

  • opakovane cinnosti pri vyvoji
  • viacere casti systemu, zavisle od 1 domenoveho modelu -> vznik inkonzistencii
  • slaba pouzivanost nastrojov pre dizajn SW

Riesenie

  • automatizacia
  • generativne programovanie
  • podporne nastroje

Postup

  1. analyza aktualneho stavu systemu BFI
  2. refaktorizacia s cielom zlepsenia modularity
  3. rozsirenie jazyka BFI
    1. podpora enumeracnych typov
    2. kontrola obsahu atributov (field constraints)
  4. zlepsenie modularity riesenia
  5. pridanie funkcionalit
    • podpora DB migracii
    • podpora DevOps
    • podpora kontroly pristupu k zdrojom
  6. zlepsenie pouzitelnosti riesenia
  7. vytvorenie podpornych nastrojov

BFI - Backend-frontend interface

Tvorba zdrojovych suborov

                model Task
                  name:string label="Task name"  matches="[A-Za-z ]{1,20}" 
                  state:State label="Task state" 
                  categories:Category[] label="Task categories" 
                  user:User
                ....
                enum State [Todo,InProgress,Done]
                role User
                  can manage Task where .user.name: #username
                  can read Category
                  can read User where .name: #username
                ....
